HTML basics

HTML stands for hypertext markup language. Markups are tags. E.g., "b" tag makes text bold, and "em" is for italics. Void tags are tags that don't require a closing tag (that's the one with a "/" at the end of a piece of text that you want to apply your tag to).

Whitespace

Two types of tags are inline and block tags. Unlike inline tags, block tags (e.g. "p" tag I'm using for this very paragraph) actually create a box around the text they are applied to.

Stage 1: Webpages, Documents, and Structure

HTML, JavaScript, and CSS

HTML represents the structure of a webpage, JavaScript is for interactive components, and CSS adds styles.

DOM (Document Object Model)

DOM is a tree-like stucture of a page, where every tage is an element in a tree.

Everything is a Box!

It's useful to start looking at every webpage from the perspective of "seeing boxes". E.g., every header on this very page is a box, every text paragraph is a a box, etc. If I had pasted an image here, it would be its' own box, too.

The Box Model (and how to make lists!)

Every box consists of the following elements:

  • Margin (clears the area around the border)
  • Border
  • Padding (clears the area around the content)
  • Content (nucleus of the box; like this text inside it's "paragraph" box)

Also, I just accidentally learned how to make lists: you need a "ul" ("ol" for numbers) tag and an "li" (void) tag for every bullet/number. I'm happy I know how to do it now - I basically think in bullet points! Here is a link for more details/options: how to make lists.

Flexbox

If you would like to make a box flexible (for better alignment of elements on a page), you need to add "display: flex" in the CSS code. For example, I used it for a "koala" box to align the picture and its' description the way I wanted (took a while...).
A useful guide for all things flex can be found here.

Code, Test, Refine

This is a routine we all should follow while designing a page:

  1. Look for natural boxes.
  2. Look for repeates styles and semantic elements.
  3. Write HTML.
  4. Apply styles (from biggest to smallest).
  5. Fix things.
